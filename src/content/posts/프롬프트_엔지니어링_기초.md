AI를 사용하다가 가끔 말을 이해하지도 못하고 이상한 답변을 받은 경험, 해보신 적 있지 않으신가요?

기껏 시간과 돈을 써서 질문을 했는데, 원하는 답변을 받지 못할 때의 답답함을 해소하기 위해 기초적인 프롬프팅 방법을 정리해봤습니다.

## Generated Knowledge Prompting (GKP)

AI가 답변하기 전에 **참고 지식**을 먼저 만들어두고, 그 지식을 바탕으로 답하게 하는 방법입니다.

> **Generated Konwledge** : 사람이 직접 준 자료가 아니라, AI가 답하기 전에 스스로 정리해 만든 지식

언어 모델이 바로 답변을 줄 때는 배경지식이 부족하거나, 중요한 포인트를 잡지 못하는 때가 있습니다.
GKP 기법을 사용하면, 지식 정리 후 답변을 하도록 유도해 답변의 정확도를 올릴 수 있습니다.

### 활용 방안
- 답변이 너무 짧거나, 피상적인 내용으로만 구성된 경우
- 역사, 기술 등 답변을 받기 전에 정리가 필요한 주제를 다룰 때

### 예시 1) 지식 생성 -> 질문*

**1단계 - 지식 생성**
```text
세계기억력대회(World Memory Championships)의 역사와 주요 변화를 연도 흐름으로 간단히 정리해주세요.
```

**2단계 - 질문**
```text
그 대회를 처음 만든 사람(또는 창립에 핵심이 된 인물/단체)은 누군가요?
근거가 불확실하면 "불확실"이라고 표시해주세요.
```

### 예시 2) 한 번에 질문하기

```text
답변하기 전에, 운동이 건강에 미치는 효과를 5가지로 정리해주세요..
그 다음 그 정리 내용을 근거로 "왜 운동이 건강에 좋은지"를 설명해주세요.
```

## Chain-of-Thought (CoT)

복잡한 문제를 풀 때, 중간 추론 단계를 거쳐 답을 더 정확하게 만드는 방법입니다.

쉽게 말하면, AI로 하여금 **단계적**으로 문제를 풀게 유도하는 방법입니다.

### 특징
- **연속성** : 시작과 끝까지 중간 추론 단계가 논리적으로 이어져야 합니다.
- **유착성** : 사람이 읽어도 자연스럽게 이해되는 설명이 좋습니다.
- **효과성** : 중간 단계가 실제로 정답에 도움이 되어야 합니다.

### 활용 방안
- 복잡한 로직, 조건이 많은 문제
- 계획을 세우고 실행해야 하는 경우

### 예시
다음과 같은 질문을 수행하고 싶다고 가정해보겠습니다.
> 식당에 사과가 23개 있었습니다. 점심을 만들기 위해 20개를 사용했고, 6개를 더 샀습니다. 지금 사과는 몇 개일까요?

**질문**
```text
예시) 연필 10자루 중 3자루를 쓰고 5자루를 사면?
- 처음 10자루
- 3자루 사용 → 7자루
- 5자루 구매 → 12자루
답: 12자루

이제 같은 방식으로 아래 문제를 풀어줘:
(문제 본문)
```

## Self-Consistency
같은 문제를 여러 방법으로 풀어본 뒤, 가장 적합한 답을 고르는 방식입니다.

CoT와 달리 여러 번 문제를 풀고, 다수결 or 일관성이 높은 답을 선택하는 방법입니다.

### 특징
- 서로 다른 답이 나올 수 있고, 그 중 일부는 올바르지 않을 수 있습니다.
- 따라서 가장 자주 등장하는 답이나, 모순이 적은 답을 선택합니다.

### 활용 방안
- 정답률이 중요한 문제를 풀 때
- 정답이 1개로 정해진 문제를 풀 때

#### 예시
```text
이 문제를 서로 다른 방식으로 3번 풀어주세요.
각 풀이의 최종 답만 먼저 모아서 비교하고,
가장 일관된 답을 최종 답으로 선택해주세요.
```

## Prompt Chaining

큰 일을 작은 작업으로 쪼개서, 이전 답을 다음 질문의 입력으로 쓰는 방식입니다.

개인적으로 많이 활용하게 되는 방법입니다. 여러 항목으로 분할해서 작업을 진행할 때, 자주 사용합니다.

### 특징
- 복잡한 작업을 단계별로 처리하여, 답변이 안정적으로 출력됩니다.

### 활용 방안
- 긴 글을 작성할 때
- 조건이 많은 작업을 처리할 때 (예시: 기존 코드 분석 -> 리팩토링)

### 예시

**1단계**

```text
주제에 대한 핵심 목차를 6개로 만들어주세요.
```

**2단계**

```text
2번 목차 내용을 초보자 기준으로 쉽게 풀어서 500자 정도로 써주세요.
```

**3단계**

```text
방금 쓴 문단에 예시를 1개 추가해주세요.
```

## Tree of Thoughts (ToT)

AI에게 여러 역할 혹은 전문가를 부여해서 다양한 해법을 탐색하고, 중간에 평가 및 수정을 거쳐 결론을 내는 방법입니다.

저는 심도있는 문제를 다루거나, 설계가 필요한 문제에 대한 실마리를 찾을 때 자주 사용하는 방법입니다.

### 특징
- 여러 역할 / 전문가가 서로의 의견을 반론하며 추론을 진행
- AI의 추론 과정을 실시간으로 확인 가능

### 활용 방안
- 정답이 1개가 아닌 문제 (기획, 설계 문제)
- 다양한 아이디어를 확인해야 하는 작업

### 예시
```text
서로 다른 관점 3개(예: 비용 관점/사용자 관점/기술 관점)에서 해결책을 제시해주세요.
각 관점의 약점을 서로 지적하게 한 뒤,
최종적으로 가장 균형 잡힌 결론을 고르세요.
주제: (여기에 문제)
```

## Retrieval Augmented Generation (RAG)

AI가 문서(검색/DB)를 먼저 찾아서, 해당 내용을 근거로 답을 만드는 방식입니다.

RAG는 특정 도메인에 대한 데이터를 기반으로 AI가 답변할 수 있는 방법이기에, 여러 회사에서 많이 사용하는 방법입니다.

벡터 데이터베이스, 텍스트 임베딩 모델, LLM 이 꼭 필요하기 때문에, 어느 정도의 구현 난이도가 있습니다.

### 특징
- LLM의 할루시네이션 문제를 방지
- 특정 전문 분야의 지식을 쉽게 통합

### 활용 방안
- 사내 규정, 메뉴얼 등을 문서를 기반으로 답해야 할 때
- 최신 문서를 기반으로 답해야 할 때

### 예시
```text
아래 문서 내용을 근거로 답해주세요.
- 관련 문장(근거)을 인용(짧게)하고
- 그 다음 내 질문에 답해주세요.

[문서 내용 붙여넣기]

질문: 우리 회사 환불 정책에서 "부분 환불" 조건이 뭔가요?
```

위 예시는 간단한 예시일 뿐이며, 실제 서비스에서 RAG를 사용할 때는 더욱 복잡한 처리 과정이 포함됩니다.
